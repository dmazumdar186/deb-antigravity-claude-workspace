## Real bug fixed during audit

**Fingerprint fragility on Windows CRLF**: the embed-prompts.mjs script was computing fingerprint on raw file bytes including CRLF line endings, while at-deploy time the file may have been LF. Result: spurious drift mismatches after git checkout. Fixed by normalizing CRLF→LF in both embed-prompts.mjs and the unit test before hashing. Without this fix, every fresh clone on Windows would have failed the prompt-drift unit test.
