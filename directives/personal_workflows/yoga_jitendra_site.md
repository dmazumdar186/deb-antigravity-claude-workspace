# yoga_jitendra_site

Personal website for Jitendra Kumar, Paris-based Hatha Yoga teacher. Replaces his
Google Sites placeholder at `sites.google.com/view/yogaavecjitendra/home`.
Astro static site + Cloudflare Pages (free tier).

## Goal

Ship a peaceful, traditionally-flavoured, mobile-first bilingual (FR/EN)
marketing site for Jitendra that:

- puts his real photos + real testimonials + press mention front and centre,
- honours the Indian yoga lineage (Sanskrit shloka + Ashtanga + rishi/muni
  framing),
- provides a WhatsApp-first booking path (no calendar for v1, per operator),
- feels traditional, not gym-y — through palette (cream / terracotta / sage),
  typography (Fraunces + Inter + Noto Serif Devanagari), procedural ambient
  audio (tanpura + bansuri on Raga Bhairav + formant-synthesised Om chant),
  and an inline-SVG mandala backdrop generated by GLM 5.2.

## Inputs

- `_raw_assets/` — 24 candidate files supplied by operator (17 photos + 7 review
  screenshots). **This directory is gitignored** to keep third-party faces
  outside git history.
- Site content in `src/content/*.{fr,en}.json` — every visible string, both
  languages.
- Design brief in `design_brief.md` (fed into GLM 5.2 prompts) and in
  `~/.claude/plans/https-sites-google-com-view-yogaavecjite-synthetic-sunrise.md`
  (the approved plan).

## Tools / scripts

- Astro 5 + Tailwind + Motion One — build framework (`npm run dev`, `npm run build`).
- **GLM 5.2 via `execution/modules/model_router.call_model('glm', ...,
  sensitivity="public")`** — generates the creative HeroBackdrop.astro (inline
  SVG mandala + CSS animations, ~7.5 KB). Prompt engineering + hard-cap 2 rounds
  per slice live in `generate.py`. Per `~/.claude/rules/model-tier.md` Exhibit
  C, no PII / testimonial names / phone / email is ever sent to GLM.
- Cloudflare Pages via `npx wrangler pages deploy dist/` (matches
  `portfolio_site` deploy pattern).
- Tests: `tests/acceptance_yoga_jitendra.py` (hard-fail output-acceptance gate)
  + `tests/front_door_yoga_jitendra.sh` (live URL front-door synthetic).

## Outputs

- **Public site:** https://yoga-jitendra.pages.dev (main branch).
- **Custom domain:** deferred — Jitendra will buy a `.fr`/`.com` and bind in
  the CF dashboard.
- **Build output:** `dist/` — pushed to CF Pages on every `npx wrangler pages
  deploy dist/`.

## Steps

1. `npm run build` — Astro builds `dist/index.html` and copies `public/assets/`
   images and inlined CSS.
2. `py tests/acceptance_yoga_jitendra.py` — must exit 0.
3. `npx wrangler pages deploy dist/ --project-name=yoga-jitendra --branch=main` —
   uploads incremental delta to CF Pages CDN.
4. `bash tests/front_door_yoga_jitendra.sh https://yoga-jitendra.pages.dev` —
   asserts live artifact contract holds. Must exit 0.
5. Only after 5 consecutive daily green runs on step 4 may any report describe
   the site as "shipped / live / ready" per
   `~/.claude/rules/front-door-synthetic.md`.

### Regenerating the GLM 5.2 mandala

```bash
py execution/personal_workflows/yoga_jitendra_site/generate.py
```

- Sends `design_brief` + placeholder tokens only.
- 2-round hard cap. On fail: hand-written fallback ships (documented in
  Edge Cases below).
- Cost per successful round: **~€0.03** at OpenRouter's `z-ai/glm-5.2` rates.

## Edge cases

- **Autoplay policy.** Every browser blocks `<audio>` autoplay-with-sound.
  Site works around by resuming `AudioContext` on first user gesture
  (`mousemove` + click + touch + scroll + keydown). Fresh visitor hears
  silence until they move their cursor — practically ~500 ms after load.
- **`prefers-reduced-motion`.** All animations (mandala rotation, breath
  keyframes, hero-headline word reveal, gallery hover, Om chant is audio only)
  degrade cleanly when the OS accessibility setting is on.
- **Group-session photos.** Curation avoids close-up faces of attendees
  without documented consent. If operator later warrants consent for wider
  usage, we can broaden the curation set (see `_raw_assets/`).
- **Sanskrit rendering.** Uses Google Font `Noto Serif Devanagari`. If font
  fails to load, `Fraunces` falls back — Devanagari glyphs will still render
  via the OS's native fallback fonts on macOS / iOS / Android; on Windows
  without any Devanagari font the shloka may render as boxes. This is a
  known browser limitation, not a site bug.
- **Cost cap.** GLM 5.2 slices hard-capped at 2 rounds × 3 slices × ~€0.03 =
  **~€0.20 max per full regeneration**. If OpenRouter balance is $0, the
  script falls back to Anthropic Sonnet 4.6 (~€0.30/round) — documented in
  `generate.py`.
- **Testimonial names + phone/email/address.** Kept in local
  `content/*.{fr,en}.json` files. **Never sent to GLM 5.2.** Substituted at
  Astro build time only.
- **Custom domain.** Deferred. When Jitendra buys a domain, bind in the CF
  Pages dashboard (Settings → Custom domains). No code change needed.

## Exit criteria

The project is only describable as "shipped / live / ready" once ALL of the
following hold:

1. `py tests/acceptance_yoga_jitendra.py` exits `0` — 43+ assertions green.
2. `bash tests/front_door_yoga_jitendra.sh` exits `0` against
   `https://yoga-jitendra.pages.dev` — 24+ live assertions green.
3. Steps 1-2 have both passed for **5 consecutive daily runs** (per
   `~/.claude/rules/front-door-synthetic.md`). Until day 5 the site status is
   `LIVE-PROBATIONARY: day N of 5`.
4. Jitendra has personally reviewed the FR + EN copy (proxy PM = operator).
5. RGPD / droit-à-l'image warranty from operator on any group photo showing
   identifiable non-Jitendra faces.
6. Lighthouse mobile audit: Performance ≥85, Accessibility ≥95, LCP <2.5 s.

Any FAIL on 1-3 blocks the "live" claim. Skip on 4-6 is allowed but must be
logged in the wrap-up.
