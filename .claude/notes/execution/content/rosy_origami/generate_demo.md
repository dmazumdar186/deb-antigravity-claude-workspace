# Notes — execution/content/rosy_origami/generate_demo.py

- [technical] Instaloader workaround: instaloader fails on profile-level scraping (rate-limit + login-wall) in Phase 0. The working path is per-shortcode download via `instaloader.Post.from_shortcode(L.context, shortcode)`. Profile-level `instaloader.Profile.from_username()` is blocked without a logged-in session. `fetchers/ig_instaloader.py` uses the per-shortcode path.
- [technical] `--ig-source manual` reads from `.tmp/rosy_origami/{slug}/ig_export/captions.json`. The JSON schema is: `[{type, title, caption, timestamp (ISO8601), permalink (opt), image (filename, opt)}]`. Malformed entries cause a KeyError on `entry["caption"]` — add a try/except per-entry at Phase 1 refactor.
- [technical] Flag renamed from `--tier` to `--mode` (2026-06-12, upgrade batch 2C). Values changed: `gemini` → `cheap`, `default` → `balanced`, `premium` → `premium`. The flag is parsed but currently unused in generate_demo.py — model selection happens inside composer.py. Wire `args.mode` through to composer at Phase 1.
- [technical] `call_humanizer()` at line 208 passes `--tier gemini` to the HUMANIZER subprocess. This is the humanizer's own `--tier` flag, entirely separate from generate_demo's `--mode`. Do NOT confuse the two when modifying either script.
- [technical] `--yes --three` syntax: when bootstrapping the Remotion overlay that renders newsletter sections as video (Phase 3+), the correct flag is `npx create-video@latest --yes --three`, not `--template three`. The `--three` flag (not `--template`) enables the Three.js template.
- [constraint] Output dir convention: all outputs go to `.tmp/rosy_origami/{slug}/`. The `.tmp/` prefix is gitignored. Do NOT write to `output/` or `out/` — those are used by Remotion.
- [learned] 8 files listed in the directive's Tools/Scripts table (`humanize.py`, `render.py`, `fetchers/ig_api.py`, `fetchers/ig_manual.py`, `fetchers/yt.py`, `fetchers/news.py`) were inlined into `generate_demo.py` and `composer.py` during Phase 0 rapid iteration. The directive table is out of sync with disk. At Phase 1 refactor, either create the listed files or update the table.
- [pattern] `--dry-run` skips all LLM + humanizer + Tavily calls. The pool is fetched (IG manual export is read), but compose_sections() returns placeholder text. Use this to validate config + pool loading without spending API credits.
